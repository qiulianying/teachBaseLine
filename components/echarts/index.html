1、我发现如果是在vue.js中使用到了echarts，那么，echarts的绝大部分问题，几乎都是可以通过设置组件，然后绑定key来解决

这里主要是测试echarts的组件，和实际前端学习的关系不大，业务需求，所以你学完了吃饱饭没事做可以来研究下试试

功能：点击鼠标右键添加节点

这里发现了echart的一个比较奇怪的地方，我执行代码的时候，如果将定位layout，和fixed定位公用的话，第一次执行是不会报错得， 也就是说，当我数据中直接包括的情况下，不会报错，但是我数据改变之后，重新myChart.setOption的时候，就会报错

解决方法： 也就是说，如果需要让这个数据变化之后还能生效并执行不报错，那么就需要你在数据变化之后，重新把这一块的组件完全 重新执行一遍，那么，在vue.js中，可以通过设置key；解释：在 vue.js中，具体组件默认是加载之后，不在重新加载，而是默认使用缓存页面，减少了加载和消耗，但是如果你给定义:key=time， 而组件的key的time会改变，那么这个组件就会被重新加载
