//立即执行函数：后面附带()表示对这个匿名函数的立即调用
function f() {
    console.log('函数，但是不会被调用s')

//下面就是立即执行函数的方式

js定义赋值：var 和 let

在js学习的第一课，学习到的就是定义，var是js最基本的定义方式

相对于var来说，let多了限制，才会显得更加规范，var和let都是可以用于定义 全局变量以及局部变量
不同在于：let没有变量提升，只能作用域中有效，暂时性死区以及不能重复定义

JavaScript引擎的工作方式是，先解析代码，获取所有被声明的变量，然后再一行一行的运行，这造成的结果，就是所有的变量的声明语句，都会被提升到代码的头部，这就叫做变量提升 

                //例如下面代码的执行
                console.log(aa);        //undefinde，不会报错
                var aa = 1

                //根据js工作原理相当于这个代码是这样运行的
                var aa;
                console.log(aa);
                aa = 1;     //表示变量a已声明，但还未赋值。但是变量提升只对var命令声明的变量有效，如果一个变量不是var变量声明的，就不会发生变量提升

所以这样的情况下就不是非常的规范，比如正常是结婚啪啪生小孩，变量提升会导致啪啪结婚生小孩(赋值没过去) 

//不同1：let只能在作用域中有效：更加适合for循环
{
    let bx = 6
    var ax = 6;
}
//ax = 6  bx:报错

//不能重复定义
let ax = 6; let ax = 7; //报错

//没有变量提升
console.log(ax); let ax = 6;    //报错
