页面盒模型

盒模型就是前段的一个比较基础的知识，这里有带css的知识点

属于w3c标准盒模型

margin: 用于控制元素与元素之间的距离；margin的最基本用途就是控制元素周围空间的间隔，从视觉角度上达到相互隔开的目的。

padding: 用于控制内容与边框之间的距离；

Border(边框): 围绕在内边距和内容外的边框。

Content(内容): 盒子的内容，显示文本和图像。

类似样式：
width: 200px;
height: 200px;
background: #f60;
margin: 30px;
padding: 30px;
border: 30px solid;
text-align: center;
line-height: 80px;

ie盒模型

w3c中的盒子模型的宽:包括margin+border+padding+width;
    width:margin*2+border*2+padding*2+width;
    height:margin*2+border*2+padding*2+height;
iE中的盒子模型的width:也包括margin+border+padding+width;
上面的两个宽度相加的属性是一样的。
因此我们应该选择标准盒子模型，在网页的顶部加上 DOCTYPE 声明。

iframe的其他使用方式

科普一下，使用iframe在页面中会导致页面加载熟读降低， 所以一般情况下，减少使用iframe也是一种加快页面加载的一种方式

测试iframe之间的通信：contentWindow.postMessage：参考文献

思路1：传入数据给iframe引入的html，在html中去监听这个传入的数据，根据传入数据去执行方法， contentWindow.postMessage(message, targetOrigin, [transfer]);

思路2：如果不允许修改iframe的内容时，可以直接调用iframe的方法去执行， contentWindow.('具体的方法') //contentWindow.btnClick_fp()

子页面返回当前页面的数据： 

                   //获取父页面传递的数据
                   window.addEventListener('message', function (event) {
                       console.log(event.data);
                       document.getElementById('indexCtn').innerText = event.data
                   });
                   function toReturnCtn() {
                       let num = getRndInteger(1, 20);
                       window.parent.postMessage(num);
                   }
